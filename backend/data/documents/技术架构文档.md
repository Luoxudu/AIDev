# 星辰科技 — 技术架构文档

## 系统概述

星辰科技核心产品 **StarBrain 智能决策平台** 是一套面向企业的 AI 决策支持系统，
支持数据接入、模型训练、推理部署和结果可视化的全流程管理。

平台采用微服务架构，由 12 个核心服务组成，支持 SaaS 多租户模式和私有化部署。
当前已服务超过 200 家企业客户，日均处理推理请求超过 500 万次。

## 核心微服务

### Auth Service（认证服务）

负责用户注册、登录、Token 管理和权限校验。

- **注册**：邮箱+手机号双验证，密码强度校验
- **登录**：支持密码登录、短信验证码登录、企业 SSO（对接 LDAP/SAML/OIDC）
- **Token**：JWT + Refresh Token 机制，Access Token 有效期 2 小时
- **权限**：RBAC 模型，支持角色、权限、资源三级控制
- **多租户**：基于 Tenant ID 的数据隔离

### Model Service（模型管理服务）

管理 AI 模型的注册、版本、部署和下线。

- 模型注册：支持从 MLflow 或 HuggingFace 导入模型
- 模型版本：每个模型可维护多个版本，支持版本切换和回滚
- 模型部署：一键部署到 Kubernetes 集群，自动扩缩容
- 模型评估：提供 A/B 测试框架，对比不同版本模型效果

### Inference Service（推理服务）

对外提供统一的推理接口，支持同步和异步两种模式。

- **同步推理**：适用于实时场景，P99 延迟小于 500ms
- **异步推理**：适用于批量处理，通过消息队列解耦
- **流式推理**：支持 SSE 协议的流式输出，用于聊天场景
- **多模型路由**：根据请求参数自动路由到对应的模型服务

### Train Service（训练服务）

管理模型训练任务的生命周期。

- 支持分布式训练（PyTorch DDP / DeepSpeed）
- 支持 GPU 资源调度（对接 Kubernetes GPU Operator）
- 训练过程可视化（TensorBoard 集成）
- 训练完成后自动评估并生成报告

### Data Service（数据服务）

负责数据接入、清洗、标注和版本管理。

- 支持 15+ 数据源接入（MySQL、MongoDB、Kafka、S3 等）
- 数据清洗管道：去重 → 格式化 → 质量检查 → 脱敏
- 数据标注平台：支持文本分类、实体识别、图像标注等
- 数据版本管理：基于 DVC (Data Version Control)

### Report Service（报告服务）

生成分析报告和可视化图表。

- 支持 PDF、HTML、Excel 多种格式导出
- 报告模板可自定义
- 定时报告：支持 cron 表达式配置定时生成

## 数据库设计

### 核心表结构

#### users 表

```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL REFERENCES tenants(id),
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(50) NOT NULL DEFAULT 'user',
    status VARCHAR(20) NOT NULL DEFAULT 'active',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_users_tenant ON users(tenant_id);
CREATE INDEX idx_users_email ON users(email);
```

#### models 表

```sql
CREATE TABLE models (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    tenant_id UUID NOT NULL REFERENCES tenants(id),
    name VARCHAR(200) NOT NULL,
    version VARCHAR(50) NOT NULL,
    framework VARCHAR(50) NOT NULL,
    storage_path VARCHAR(500) NOT NULL,
    status VARCHAR(20) NOT NULL DEFAULT 'registered',
    metrics JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(tenant_id, name, version)
);
```

#### inference_logs 表（分区表）

```sql
CREATE TABLE inference_logs (
    id BIGSERIAL,
    tenant_id UUID NOT NULL,
    model_id UUID NOT NULL,
    input_tokens INT NOT NULL,
    output_tokens INT NOT NULL,
    latency_ms INT NOT NULL,
    status VARCHAR(20) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
) PARTITION BY RANGE (created_at);

-- 按天分区
CREATE TABLE inference_logs_20240604
    PARTITION OF inference_logs
    FOR VALUES FROM ('2024-06-04') TO ('2024-06-05');
```

## 缓存设计

### Redis 使用场景

| 场景 | Key 模式 | TTL | 说明 |
|------|----------|-----|------|
| 用户 Session | `session:{user_id}` | 2h | 登录状态 |
| API 限流 | `ratelimit:{user_id}:{minute}` | 60s | 滑动窗口计数 |
| 模型元数据 | `model:{model_id}:meta` | 1h | 减少 DB 查询 |
| 热点数据 | `hotdata:{type}:{key}` | 5min | 高频查询缓存 |
| 任务状态 | `task:{task_id}:status` | 24h | 异步任务状态 |
| 分布式锁 | `lock:{resource}` | 30s | 防止并发冲突 |

### 缓存策略

- **Cache-Aside**：读时先查缓存，Miss 后查 DB 并回写缓存
- **Write-Through**：写操作同步更新缓存和 DB
- **缓存穿透防护**：使用 Bloom Filter + 空值缓存
- **缓存雪崩防护**：TTL 加随机偏移（±10%）

### 告警规则

```yaml
groups:
  - name: startech_alerts
    rules:
      - alert: HighLatency
        expr: histogram_quantile(0.99, http_request_duration_seconds) > 1
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "P99 延迟超过 1 秒"

      - alert: HighErrorRate
        expr: rate(http_requests_total{status="500"}[5m]) > 0.05
        for: 2m
        labels:
          severity: critical
        annotations:
          summary: "5xx 错误率超过 5%"

      - alert: GPUUtilizationLow
        expr: gpu_utilization_percent < 10
        for: 30m
        labels:
          severity: info
        annotations:
          summary: "GPU 使用率过低，可能存在资源浪费"
```

## 安全架构

### 网络安全

- 所有服务运行在 VPC 内网，仅 API Gateway 暴露公网入口
- 服务间通信使用 mTLS
- Kubernetes NetworkPolicy 限制 Pod 间通信
- WAF 防护常见 Web 攻击（SQL 注入、XSS、CSRF）

### 数据安全

- 数据库开启 TDE（透明数据加密）
- 敏感字段（密码、手机号、身份证号）使用 AES-256 加密存储
- 日志中自动脱敏敏感信息
- 数据库备份加密后存储至异地机房

### 合规认证

- 已通过 ISO 27001 信息安全管理体系认证
- 已通过国家信息安全等级保护三级认证
- 已通过 SOC 2 Type II 审计

## 灾备与容灾

### 备份策略

| 数据类型 | 备份频率 | 保留期 | 存储位置 |
|----------|----------|--------|----------|
| 数据库全量 | 每日 02:00 | 30 天 | 异地 S3 |
| 数据库增量 | 每 6 小时 | 7 天 | 同城备库 |
| 文件存储 | 每日 03:00 | 90 天 | 异地 S3 |
| 配置信息 | 每次变更 | 永久 | Git + S3 |

### 容灾等级

- **RPO（恢复点目标）**：小于 1 小时（数据库）
- **RTO（恢复时间目标）**：小于 30 分钟（核心服务）
- **多活**：北京和上海双机房，通过 DNS 智能解析实现流量切换

## 性能优化

### 已实施的优化措施

1. **数据库**：读写分离（1 主 2 从），慢查询监控，连接池（pgbouncer）
2. **缓存**：多级缓存（本地 Caffeine + Redis），热点数据预热
3. **API**：响应 gzip 压缩，分页 + 字段过滤，批量接口
4. **推理**：模型量化（INT8）、算子融合、动态批处理（Dynamic Batching）
5. **前端**：代码分割、CDN 加速、虚拟滚动、资源预加载
